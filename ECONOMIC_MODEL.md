# THE RECORD: Economic Model - Answering the Core Question

## The Problem You Identified

**Question:** "Why would someone pay anything to make a claim that their house is on fire, much less someone else's house? Where does the system start off having money to pay people for reputation slashing?"

**Answer:** They wouldn't. And the system shouldn't require it.

This is the fundamental flaw in many "stake-based" systems. **THE RECORD does NOT require payment to submit signals.**

---

## The Actual Economic Model

### Core Principle: **Free to Signal, Cost to Lie**

```
SUBMITTING A SIGNAL:
├─ Cost: $0 (FREE)
├─ Requirement: Humanity score > 30 (proof you're human)
├─ Risk: If proven false, you lose influence (NOT money)
└─ Reward: If validated, you gain influence

INFLUENCE ≠ MONEY
├─ Influence: Your weight in future decisions
├─ Cannot be bought or sold
├─ Earned through accurate reporting
└─ Lost through false reporting
```

### Why This Works

**Example: House Fire**

```
1. USER REPORTS: "My house is on fire"
   ├─ Cost to submit: $0
   ├─ System asks: Is this urgent?
   └─ Creates L1 signal immediately

2. SYSTEM RESPONDS:
   ├─ Alerts nearby users to witness
   ├─ Pings professional validators (fire dept)
   ├─ Creates priority cluster
   └─ Emergency services contacted

3. VALIDATION:
   ├─ Other users attest: "Yes, I see smoke"
   ├─ Fire department confirms
   └─ Reporter gains +100 influence

4. IF FALSE:
   ├─ No smoke, no fire
   ├─ Reporter loses -100 influence
   ├─ Future reports weighted less
   └─ Repeated false reports = shadowban
```

**Key insight:** The cost of lying is **reputation**, not money.
